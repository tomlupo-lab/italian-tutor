# Core Italian Sentence Patterns

## Purpose

This document defines a compact set of reusable Italian sentence patterns for fast functional progress.

Related documents:

- [fast-italian-roadmap.md](/home/twilc/projects/italian-tutor/italian-tutor/docs/fast-italian-roadmap.md)
- [curriculum-content-model.md](/home/twilc/projects/italian-tutor/italian-tutor/docs/curriculum-content-model.md)
- [content-review-checklist.md](/home/twilc/projects/italian-tutor/italian-tutor/docs/content-review-checklist.md)

These patterns are intended to support:

- chunk-first SRS design
- early exercise generation
- mission preparation and calibration hints
- backend inventory audits

They are not a complete grammar syllabus. They are the highest-yield production templates that let a learner build many useful sentences quickly.

## How To Use This Document

Each pattern includes:

- the core template
- what it is useful for
- an approximate phase
- example substitutions
- notes on caution or scope when needed

Phases map to the fast roadmap in [fast-italian-roadmap.md](/home/twilc/projects/italian-tutor/italian-tutor/docs/fast-italian-roadmap.md).

## Phase Reference

- Phase 1: Survival Italian
- Phase 2: Real Daily Conversations
- Phase 3: Functional Independence

## Pattern Selection Rules

These patterns were chosen because they are:

- frequent in real conversation
- easy to reuse with substitution
- useful across many domains
- strong enough to support speaking early

Patterns were avoided or downgraded if they:

- require advanced grammar too early
- are too narrow or literary
- look simple but hide unstable grammar demands

## Core Patterns

## 1. Identity and basic description

### Pattern ID

`identity_essere`

### Pattern

```text
Sono + professione / nazionalità / aggettivo
```

### Use

- introducing yourself
- basic description
- simple states

### Phase

- Phase 1

### Examples

```text
Sono polacco.
Sono stanco.
Sono ingegnere.
```

## 2. Duration with `da`

### Pattern ID

`duration_da`

### Pattern

```text
verbo + da + tempo
```

### Use

- saying how long something has been true

### Phase

- Phase 1 to Phase 2

### Examples

```text
Sono qui da due giorni.
Studio italiano da un anno.
Lavoro qui da sei mesi.
```

### Note

This is better treated as a duration frame than as a fixed `sono + da + tempo` pattern.

## 3. Location

### Pattern ID

`location_essere`

### Pattern

```text
Sono + a / in + luogo
```

### Use

- saying where you are

### Phase

- Phase 1

### Examples

```text
Sono a casa.
Sono in ufficio.
Sono al supermercato.
```

## 4. Movement and destination

### Pattern ID

`movement_vado`

### Pattern

```text
Vado + a / in + luogo
```

### Use

- saying where you are going

### Phase

- Phase 1

### Examples

```text
Vado al lavoro.
Vado in Italia.
Vado a Roma.
```

## 5. Wants

### Pattern ID

`want_voglio`

### Pattern

```text
Voglio + verbo / oggetto
```

### Use

- expressing direct wants

### Phase

- Phase 1

### Examples

```text
Voglio mangiare.
Voglio un caffè.
Voglio dormire.
```

### Note

Useful and frequent, but less polite in service contexts than `vorrei`.

## 6. Polite requests and preferences

### Pattern ID

`polite_request_vorrei`

### Pattern

```text
Vorrei + oggetto / verbo
```

### Use

- ordering
- requesting
- expressing a polite wish

### Phase

- Phase 1

### Examples

```text
Vorrei un cappuccino.
Vorrei parlare con il direttore.
Vorrei prenotare un tavolo.
```

## 7. Ability and permission

### Pattern ID

`ability_posso`

### Pattern

```text
Posso + verbo
```

### Use

- asking permission
- asking about possibility

### Phase

- Phase 1

### Examples

```text
Posso entrare?
Posso pagare con carta?
Posso sedermi qui?
```

## 8. Obligation and necessity

### Pattern ID

`obligation_devo`

### Pattern

```text
Devo + verbo
```

### Use

- saying what you need or have to do

### Phase

- Phase 1

### Examples

```text
Devo lavorare.
Devo studiare.
Devo partire presto.
```

## 9. Needs

### Pattern ID

`need_ho_bisogno_di`

### Pattern

```text
Ho bisogno di + oggetto / infinito
```

### Use

- asking for help
- stating a need

### Phase

- Phase 1 to Phase 2

### Examples

```text
Ho bisogno di aiuto.
Ho bisogno di dormire.
Ho bisogno di un dottore.
```

## 10. Likes

### Pattern ID

`like_mi_piace`

### Pattern

```text
Mi piace / Mi piacciono + oggetto
```

### Use

- stating preferences
- talking about taste and interests

### Phase

- Phase 1 to Phase 2

### Examples

```text
Mi piace il vino.
Mi piacciono i viaggi.
Mi piace questa città.
```

## 11. Preferences

### Pattern ID

`preference_preferisco`

### Pattern

```text
Preferisco + oggetto / infinito
```

### Use

- comparing options
- expressing a stable preference

### Phase

- Phase 2

### Examples

```text
Preferisco il treno.
Preferisco lavorare da casa.
Preferisco andare a piedi.
```

## 12. Past experience

### Pattern ID

`past_ho_participio`

### Pattern

```text
Ho + participio passato
```

### Use

- saying what happened
- reporting completed events

### Phase

- Phase 2

### Examples

```text
Ho mangiato.
Ho visto un film.
Ho lavorato molto.
```

### Note

This is only one major branch of `passato prossimo`. Learners will later also need common `essere` patterns such as `sono andato`.

## 13. Plans and intention

### Pattern ID

`plan_penso_di`

### Pattern

```text
Voglio / Penso di + verbo
```

### Use

- stating plans
- expressing intention

### Phase

- Phase 2 to Phase 3

### Examples

```text
Voglio viaggiare.
Penso di studiare di più.
Penso di cambiare lavoro.
```

## 14. Future plans

### Pattern ID

`future_simple`

### Pattern

```text
futuro semplice + complemento
```

### Use

- talking about future actions
- making plans more explicit

### Phase

- Phase 3

### Examples

```text
Andrò in Italia.
Farò sport.
Studierò di più.
```

### Note

This is more general and more useful than memorizing a narrow pattern like `andrò / farò + oggetto`.

## 15. Explanations

### Pattern ID

`explanation_perche`

### Pattern

```text
... perché + frase
```

### Use

- giving reasons
- connecting two ideas

### Phase

- Phase 1 to Phase 2

### Examples

```text
Studio italiano perché mi piace.
Sono stanco perché ho lavorato.
Non esco perché devo studiare.
```

## 16. Opinions with low grammar overhead

### Pattern ID

`opinion_secondo_me`

### Pattern

```text
Secondo me + frase
```

### Use

- giving opinions
- softening disagreement

### Phase

- Phase 2 to Phase 3

### Examples

```text
Secondo me è troppo caro.
Secondo me è una buona idea.
Secondo me dobbiamo aspettare.
```

## 17. Basic connected description

### Pattern ID

`description_e_aggettivo`

### Pattern

```text
è + aggettivo
```

### Use

- describing objects, places, ideas, situations

### Phase

- Phase 1

### Examples

```text
È interessante.
È troppo caro.
È molto buono.
```

## 18. Relative clause starter

### Pattern ID

`relative_clause_che`

### Pattern

```text
nome + che + frase
```

### Use

- building longer connected sentences
- identifying people and things

### Phase

- Phase 3

### Examples

```text
Cerco un appartamento che costa meno.
Ho un amico che vive a Milano.
È un film che mi piace molto.
```

## Advanced-Caution Pattern

## 19. Opinion with `penso che`

### Pattern ID

`opinion_penso_che`

### Pattern

```text
Penso che + frase
```

### Use

- expressing more developed opinions

### Phase

- Phase 3, with caution

### Examples

```text
Penso che sia interessante.
Penso che sia difficile.
```

### Note

This pattern is common and important, but it often brings in subjunctive usage in standard Italian. It should not be treated as a Phase 1 minimal-core structure.

## Priority Ranking

If the app must prioritize a smaller subset, start with these first:

1. `Sono + ...`
2. `Sono + a / in + luogo`
3. `Vado + a / in + luogo`
4. `Vorrei + ...`
5. `Posso + verbo`
6. `Devo + verbo`
7. `Ho bisogno di + ...`
8. `Mi piace / Mi piacciono + ...`
9. `Ho + participio passato`
10. `... perché + frase`

Then add:

11. `Preferisco + ...`
12. `Voglio / Penso di + verbo`
13. `Secondo me + frase`
14. `futuro semplice + complemento`
15. `nome + che + frase`

## Mission Readiness Guidance

These patterns should guide what mission tiers are likely to assume.
They are best used as preparation and content-design hints, not as hard access rules.

### A1-leaning mission language

Mostly rely on:

- `identity_essere`
- `location_essere`
- `movement_vado`
- `polite_request_vorrei`
- `ability_posso`
- `obligation_devo`
- `need_ho_bisogno_di`
- `like_mi_piace`
- `explanation_perche`

### A2-leaning mission language

Can expand with:

- `past_ho_participio`
- `preference_preferisco`
- `plan_penso_di`
- `opinion_secondo_me`

### B1-leaning mission language

Can expand with:

- `future_simple`
- `relative_clause_che`
- `opinion_penso_che`

Mission difficulty should be judged by the total dependency set, not by one labeled pattern in isolation.

## Recommended SRS Formats

The best review types for these patterns are:

1. fill-in template
2. translation into Italian
3. substitution drill
4. audio recognition

### Fill-in example

Front:

```text
___ bisogno di aiuto.
```

Back:

```text
Ho
```

### Translation example

Front:

```text
I want to travel to Italy.
```

Back:

```text
Voglio viaggiare in Italia.
```

### Substitution example

Pattern:

```text
Posso + verbo
```

Practice substitutions:

- entrare
- pagare
- sedermi
- parlare
- prenotare

## Product Guidance

These patterns should be used as a quality standard for early content.

### SRS

- prefer chunks built from these patterns
- avoid early cards that are only isolated nouns unless strongly justified
- avoid back sides that contain metadata instead of real meaning

### Exercise generation

- early drills should recycle these patterns aggressively
- new content should vary domains while keeping syntax familiar
- generated examples should stay short and reusable

### Mission gating

- a mission should not assume language beyond these patterns unless it is explicitly mapped to a later phase
- A1 missions should rely mostly on the top-priority patterns
- A2 missions can combine these patterns with `passato prossimo`, pronouns, and comparatives
- B1 missions can expand into future, relative clauses, and more connected explanations

## Summary

These sentence patterns are the core production templates for fast Italian progress.

They should be treated as:

- the primary early speaking toolkit
- the preferred base for chunk-first SRS
- a practical constraint on what early missions are allowed to demand

They are not the whole language, but they are the right first layer for building useful speech quickly.
